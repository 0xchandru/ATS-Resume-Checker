# Project Goal
Complete redesign of the ATS Resume Checker to match Jobscan's premium, professional layout. Implement a dark-first theme with a light mode toggle, a Jobscan-style sidebar + main content layout, tabbed results, and significant backend algorithm improvements to exceed the quality of platforms like Naukri and LinkedIn.

# UI Design & Layout Idea
1. **Core Architecture**: Move away from a centered top-nav layout to a **fixed left sidebar + scrollable main content** layout, mimicking professional tools like Jobscan.
2. **Theme**: Deep navy dark mode by default with electric blue primary accents. Glassmorphism cards (backdrop-blur) and smooth theme transitions.
3. **Left Sidebar (`ResultsSidebar.tsx`)**: 
   - Large, glowing "Match Rate" donut chart.
   - "Upload & Rescan" primary button.
   - "✨ AI Optimize" secondary button.
   - 5 clickable Category Progress Bars with issue counts: Searchability, Hard Skills, Soft Skills, Recruiter Tips, Formatting.
4. **Main Content (`ResultsTabs.tsx`)**:
   - Tabbed view: **Resume Report** | **Job Description**.
5. **Resume Report Sections**:
   - `SearchabilitySection.tsx`: Checks for Contact Info, Professional Summary, standard Section Headings, and Job Title match. Shows ✅/❌ for each.
   - `HardSkillsSection.tsx`: Skills comparison with rounded chips. A "Highlighted Skills" tab showing side-by-side Resume vs JD text with matched skills highlighted in green and missing in red.
   - `SoftSkillsSection.tsx`: Dedicated analysis for soft skills (communication, leadership, etc.).
   - `RecruiterTipsSection.tsx`: Actionable tips on measurable results (quantified bullets), action verb quality, resume length, and file format.
   - `FormattingSection.tsx`: Checks for multi-column layouts, tables, fonts, and page setup.
6. **Job Description Tab (`JobDescriptionTab.tsx`)**:
   - Full JD text with highlighted matched/missing keywords.
   - An "Edit" button to update the JD text and trigger a rescan.

# Backend Alterations Needed
1. **Keyword Extraction & Matching (`extractor.py`, `matcher.py`)**:
   - Implement hard vs. soft skill classification using a predefined set of soft skills.
   - Match hard skills and soft skills separately and return them in distinct lists.
2. **New Engines (`searchability_checker.py`, `recruiter_tips.py`)**:
   - Create logic to check for the presence of email, phone, and standard section headers.
   - Create logic to calculate the percentage of experience bullets containing quantified metrics (numbers, %, $, etc.).
   - Evaluate resume word count and action verb strength.
3. **Scoring Engine (`scorer.py`)**:
   - Refactor the scoring system to output 5 specific category scores that map directly to the frontend sidebar: `searchability`, `hard_skills`, `soft_skills`, `recruiter_tips`, and `formatting`.
   - Include an `issues_to_fix` count for each category.
4. **API Router (`analyze.py`)**:
   - Update the `/analyze` endpoint to return the new structured data (category scores, separated hard/soft skills, searchability results, recruiter tips).
