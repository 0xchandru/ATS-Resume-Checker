Target: Better than Jobscan, SkillSyncer, LinkedIn Resume, Indeed, and Naukri.
Product: Upload a resume (PDF/DOCX) + paste a job description → get an ATS score,
missing keyword report, formatting diagnosis, section analysis, and recruiter-grade
improvement suggestions — powered by ESCO + O*NET knowledge bases under the hood.

=== 5-LAYER ATS MATCHING PIPELINE ===

Every keyword match traverses these layers in order. Stop at first match.

Layer 0 — ALIAS RESOLUTION  (confidence=1.0, match_layer="alias")
  Bulk SQL: SELECT canonical_name FROM kb_skill_aliases WHERE alias_normalized IN (batch)
  Resolves: "ML"→"Machine Learning", "k8s"→"Kubernetes", "JS"→"JavaScript"
  Must run BEFORE any fuzzy/semantic matching.

Layer 1 — EXACT MATCH  (confidence=1.0, match_layer="exact")
  Compare normalize(resume_term) == normalize(jd_term)
  Bulk comparison — not a loop.

Layer 2 — KB LOOKUP  (confidence=0.98, match_layer="kb_lookup")
  Bulk SQL: SELECT canonical_name FROM kb_skills WHERE normalized IN (batch)

Layer 3 — FUZZY MATCH  (match_layer="fuzzy", confidence=score/100)
  rapidfuzz.process.cdist() — batch, not a loop.
  Algorithm: token_set_ratio. Threshold: 82.
  Apply only to terms that failed layers 0–2.

Layer 4 — SEMANTIC MATCH  (match_layer="semantic", confidence=cosine_score)
  sentence-transformers all-MiniLM-L6-v2, lazy-loaded singleton.
  Encode all unmatched terms in a single batch call.
  Cosine similarity threshold: 0.72.
  Apply only to terms that failed layers 0–3.

Each matched keyword in API response includes:
  keyword, normalized_form, match_layer, matched_form, kb_source,
  category, domain, jd_frequency, jd_importance, jd_occurrence_count

=== ATS SCORING ENGINE ===

Weights (empirical — from kb_jd_frequency data, not hardcoded per-skill):
  keyword_match:        0.35
  semantic_relevance:   0.25
  section_completeness: 0.15
  format_compliance:    0.15
  impact_quantification: 0.10

keyword_match sub-scoring:
  Each matched keyword: contribution = jd_frequency * match_confidence * layer_multiplier
  Layer multipliers: alias=1.0, exact=1.0, kb_lookup=0.98, fuzzy=0.85, semantic=0.75
  Missing keyword penalty: proportional to its jd_frequency score

semantic_relevance:
  TF-IDF + cosine similarity between full resume text and full JD text (sklearn)
  Also use spaCy doc.similarity() as secondary signal; average both.

section_completeness:
  Count detected sections vs EXPECTED_SECTIONS list
  Score = (detected_count / expected_count) * 100

format_compliance:
  Start at 100. Deduct: critical=-20, warning=-8, info=-2 per issue found.
  Floor at 0.

impact_quantification:
  Parse experience section bullet points (lines starting with action verb or dash/bullet char)
  quantified_bullet = contains [0-9]+[%$xX]? or dollar sign or ratio or "X users/clients/etc"
  score = (quantified_count / total_bullets) * 100
  Additional penalty: weak verbs from kb_action_verbs strength=1 reduce this score

Grade mapping:
  95–100: A+  |  90–94: A  |  85–89: B+  |  75–84: B
  65–74: C+   |  50–64: C  |  35–49: D   |   0–34: F

=== ATS CAREER SIGNALS MODULE ===
(Surfaces as "Career Level Match" and "Role Gap Analysis" in the UI — not "occupational intelligence")

Seniority detection (both JD and resume):
  JD: extract title → query kb_job_titles → seniority_level; extract r'\b(\d+)\+?\s*(?:years?|yrs?)\b'
  Resume: all job titles held via spaCy NER + pattern → query kb_job_titles; majority vote
  Output: jd_seniority, resume_seniority, seniority_match bool, gap_explanation string

Role taxonomy matching (O*NET occupation matching):
  Match JD keywords against kb_onet_occupations.typical_skills — highest overlap = matched occupation
  Skills in typical_skills but absent from resume = "occupation-expected missing skills"
  These rank HIGHER priority than regular JD-missing skills in feedback

Company & university recognition:
  spaCy ORG entities from resume → normalize → query kb_company_aliases → kb_companies
  spaCy ORG/edu section text → normalize → query kb_universities
  Return: recognized companies with tier, recognized universities with country

JD Quality Score (unique feature — no competitor has this):
  Query kb_jd_frequency WHERE role_category = detected AND frequency > 0.85
  Find high-frequency keywords absent from the submitted JD
  jd_normality_score = 100 * (present_high_freq / total_high_freq)
  If jd_normality_score < 70: alert user that this JD is atypical — consider adding standard keywords anyway

=== CYBERSECURITY VERTICAL ===

Auto-activate if: overlap(JD keywords, kb_cyber_skills high-precision) > 15%

When active, add cybersecurity_analysis block:
  detected_certs: use kb_cert_patterns regex patterns (not NER) against resume text
  missing_critical_certs: kb_certifications entries typical for role, absent from resume
  ranked_skills_found: kb_cyber_skills matches sorted by rank

=== ACTION VERB QUALITY ANALYSIS ===

For each line in experience section:
  Extract first token → normalize → query kb_action_verbs
  Categorize: leadership / technical / analytical / communication / generic
  Strength: 1=weak (worked, helped, assisted), 2=medium (managed, developed), 3=strong (architected, spearheaded)

Output:
  verb_profile: {leadership:N, technical:N, analytical:N, communication:N, generic:N}
  weak_verbs_found: [{verb, context_sentence, strength}]
  strong_verb_ratio: float
  seniority_verb_alignment: bool (senior role → expects leadership verbs)
  suggestions: ["Replace 'worked' with 'engineered', 'implemented', or 'built'"]

=== SECTION DETECTION ===

Three-method cascade (all three run; highest-confidence result wins per section):
  Method 1: regex against kb_section_patterns for each line
  Method 2: pdfplumber char data — font size jump ≥ 2pt on a line = likely header (PDF only)
  Method 3: spaCy sentence boundary + capitalization heuristic (fallback)

Expected sections (in order): contact_info, summary, experience, education, skills,
  projects, certifications, awards, languages, volunteer, publications

Ordering score: compare detected sequence to ideal. For candidates with < 2 years
  experience: education-first order is acceptable. Detect experience year span to decide.

Flags: section word_count < 30 = "sparse"; word_count > 600 = "too long"
  Missing summary, experience, or skills = critical flag in feedback

=== FORMATTING ANALYSIS ===

PDF (pdfplumber primary, PyMuPDF fallback):
  Fonts: pdfplumber char-level → unique (fontname, size) pairs
    >3 distinct fonts: severity=warning | 1–2: severity=pass
  Columns: cluster text block x-positions; ≥2 non-overlapping clusters = multi-column
    Multi-column: severity=critical (ATS cannot parse columns reliably)
  Tables: pdfplumber.find_tables() → any table: severity=critical
  Images: PyMuPDF page.get_images() → any image: severity=warning
  Header/footer: text in top 8% or bottom 8% of page height: severity=info
  Special chars: Unicode outside ASCII 32–126: severity=warning, list the chars

DOCX (python-docx):
  Tables: doc.tables → any table: severity=critical
  Text boxes: XML scan for txbxContent: severity=critical
  Header/footer: doc.sections[0].header.paragraphs: severity=info
  Images: doc.inline_shapes: severity=warning
  Style count: unique style names used

=== FEEDBACK ENGINE ===

Generate priority-sorted suggestion cards: critical → important → nice_to_have

Critical triggers:
  - seniority_match=false AND level gap ≥ 2
  - jd_importance="critical" keyword missing from resume
  - Any format severity=critical
  - overall_score < 50

Important triggers:
  - impact_quantification score < 50
  - Missing "summary" or "experience" section
  - jd_importance="high" keyword missing
  - strong_verb_ratio < 0.6
  - occupation-expected missing skills (O*NET sourced)

Nice-to-have triggers:
  - Individual weak verbs
  - Keyword density warnings (count > KEYWORD_DENSITY_THRESHOLD)
  - Missing optional sections

Bullet rewrite suggestions (NO external LLM — use templates only):
  Find 3 lowest-impact bullets: weakest verb + no numeric metric
  Template: "[Strong verb] [what you did] by [how much], resulting in [outcome metric]"
  Example output:
    original:  "Worked on improving system performance"
    suggested: "Reduced API latency by 40% by implementing Redis caching, improving p95 from 800ms to 480ms"

=== FULL API SPECIFICATION ===

  Body: multipart/form-data { file: PDF|DOCX, job_description: string }

=== ENGINE IMPLEMENTATION RULES ===

engine/parser.py:
  PDF: pdfplumber primary — text per page, char-level data (x, y, fontname, size), tables
       PyMuPDF fallback — also extract page.get_images() list
  DOCX: python-docx — paragraphs + heading styles + table cells + inline_shapes + header/footer
  Return: { raw_text, pages[], page_count, file_type, metadata:{ fonts_used, font_count,
            has_tables, has_images, has_header_footer, table_data, char_data } }

engine/extractor.py — extract_keywords(text, top_n=50):
  1. spaCy tokenize → noun_chunks + named entities
  2. Bulk SQL: kb_skills WHERE normalized IN (all normalized tokens)
  3. Bulk SQL: kb_skill_aliases WHERE alias_normalized IN (all normalized tokens)
  4. YAKE(text, top_n) → candidate phrases
  5. KeyBERT(text, top_n) — use "en" model, not sentence-transformers
  6. Merge + deduplicate via alias resolution
  7. Attach: category from kb_skills, jd_frequency from kb_jd_frequency
  8. Return structured list with source attribution per keyword

engine/matcher.py:
  All 5 layers as specified.
  normalize(): lowercase, strip punctuation (keep hyphens + dots in version strings), collapse whitespace.
  Bulk alias lookup: single SQL IN query for all terms.
  Bulk exact lookup: single SQL IN query.
  Fuzzy: rapidfuzz.process.cdist() — not a loop.
  Semantic: single batch encode call for all unmatched terms.
  Track per-layer match counts for sub_scores.match_breakdown.

engine/career_analyzer.py:
  detect_seniority(text, context):
    spaCy NER ORG+PERSON heuristic + kb_job_titles regex patterns → extract titles
    Query kb_job_titles for each → seniority_level; majority vote
    JD only: also extract r'\b(\d+)\+?\s*(?:years?|yrs?)\b'
    Return { inferred_level, confidence, signals_found, years_mentioned }
  match_onet_occupation(keywords):
    For each occupation: score = overlap(keywords, typical_skills) / len(typical_skills)
    Return top 3 with scores
  analyze_jd_normality(jd_keywords, role_category):
    Query kb_jd_frequency WHERE role_category=? AND frequency>0.85
    Identify absent high-frequency keywords
    normality_score = 100 * (present / total)

engine/feedback_engine.py:
  No external LLM calls. Template-based bullet rewrites only.
  Priority sort: critical → important → nice_to_have.
  Trigger conditions exactly as specified in FEEDBACK ENGINE section above.
  Bullet rewrite: find 3 lowest-impact bullets (strength=1 verb + no numeric pattern)
  Apply template: "[Strong verb] [what] by [metric], resulting in [outcome]"
